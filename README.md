# 🤖 SvyaklyaBot — Telegram Bot with Attitude

**A Spring Boot-powered Telegram bot that registers users in MySQL, talks back with inline keyboards, rolls random numbers, tells jokes no one asked for, and lets the owner broadcast messages to every user who ever pressed /start. Logs everything. Roasts everyone.**

---

## 🎮 What This Bot Actually Does

| Command | Behavior |
|---------|----------|
| `/start` | Registers user in MySQL (chatId, name, username, timestamp). Greets with legendary "52!" |
| `/help` | Returns a manually crafted help text with Russian humor |
| `/mydata` | Shows stored user info from the database |
| `/randomjoke` | Returns a random string from a hardcoded array (includes OwO faces) |
| `/roll` | Inline keyboard duel — "ЛетсГоу" vs "Ненене" — random numbers, bot vs user |
| `/authorsend (text)` | Anonymous feedback sent directly to the owner's chatId |
| `/send (text)` | Owner-only mass broadcast to ALL users in the database |
| **Fallback** | Any unknown text gets: "Ты видел какие у меня огромные команды? ЙООУ??" |

## 🧠 Design Highlights

| Feature | Implementation | Why It's Cool |
|---------|---------------|---------------|
| **Auto-registration** | `registerUser()` on `/start` — checks `findById` before insert | No duplicate users, clean data |
| **Mass broadcast** | `/send` parses message, iterates `userRepository.findAll()`, sends individually | Admin tool without a dashboard |
| **Owner-only commands** | `config.getOwnerId() == chatId` guard | No one else can spam your users |
| **Inline keyboard duel** | `YES_BUTTON` / `NO_BUTTON` callback — edits message in place | Interactive, not just text replies |
| **File-rolling logging** | Logback `SizeAndTimeBasedRollingPolicy` — 10MB per file, 60 days, 1GB total cap | Production-grade log management |
| **Bot commands menu** | `SetMyCommands` on initialization | Native Telegram command autocomplete for users |

## 🗄️ Database Schema

```sql
-- Auto-generated by Hibernate ddl-auto=update
CREATE TABLE tgusers (
    chat_id BIGINT PRIMARY KEY,
    first_name VARCHAR(255),
    last_name VARCHAR(255),
    username VARCHAR(255),
    registered_at TIMESTAMP
);
```

- **`chat_id`** — natural primary key from Telegram (no surrogate needed).
- **`registered_at`** — `Timestamp` from `System.currentTimeMillis()` at registration moment.
- **`ddl-auto=update`** — dev convenience. For production: switch to `validate` + Flyway migrations.

## 🚀 Quick Run

```bash
# 1. Clone
git clone https://github.com/NeironEssensive/svyaklya-bot.git
cd svyaklya-bot

# 2. Create MySQL database
mysql -u root -p -e "CREATE DATABASE my_db;"

# 3. Configure application.properties
#    - bot.token = YOUR_BOT_TOKEN
#    - bot.owner = YOUR_TELEGRAM_CHAT_ID
#    - spring.datasource.password = your_password

# 4. Build & run
mvn clean package
java -jar target/demo-telegram-bot.jar
```
